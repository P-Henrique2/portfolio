FEATURED WORK

One project, built all the way through.

AppointerApp

An appointment scheduling app for businesses — clients book online, owners manage availability and their day from one dashboard.

Live Demo Code

Problem

Small businesses lose time and bookings to phone calls, double-booked slots, and manual calendar juggling across time zones.

Solution

A self-serve booking flow for clients, backed by an owner-facing dashboard for managing available slots, reminders, and daily schedule at a glance.

Slot creation & availability rules Client self-booking Email reminders Calendar dashboard Authentication
Version control

Git-based workflow, so changes ship without breaking what's already live.

APIs

REST endpoints drive every real action — creating slots, booking, confirming.

Hosting & deployment

Deployed and live on Vercel, with automatic deploys on every push.

Security

Server-side validation and auth checks on every request — never trusting client state.

Rate limiting

Limits on booking and auth routes to block abuse and brute-force attempts.

Caching

Cached reads on high-traffic routes keep the dashboard fast under load.

Error tracking

Errors are logged and surfaced, not silently swallowed in production.

Hard problems I solved

Timezone conflicts Normalized all slot times to a single source of truth so bookings display correctly regardless of client or business timezone.
Double-booking Added constraint checks so two clients can never claim the same slot, even under concurrent requests.
Calendar UI Built a day/week view that stays readable with a dense schedule, without a heavy calendar library.
State management Kept booking state in sync across the client view and owner dashboard without stale or conflicting data.
API design Structured endpoints around actions owners and clients actually take, not raw database tables.
